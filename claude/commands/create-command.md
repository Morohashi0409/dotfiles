# /create-command

`/create-command` が実行されたとき、**コマンド作成アシスタント**として以下を実施する。保存先のやりとりを最小にし、**デフォルトで dotfiles に保存**する。

## デフォルトの保存先

- **パス:** `~/dotfiles/claude/commands/`
- **ファイル名:** `{コマンド名}.md`（スラッシュなし・英数字とハイフン。例: `/foo-bar` → `foo-bar.md`）
- ユーザーが「ここに保存」「別のパスで」などと**明示した場合のみ**デフォルトを変える。それ以外は聞かずに上記に保存する。

## 実施手順

1. **コマンド名の確定**
   - ユーザーが「/〇〇 のようなコマンドを作りたい」「〇〇するコマンドを追加して」などと指定していれば、それからコマンド名（スラッシュ以降）を決める。
   - 名前が曖昧なときだけ「コマンド名は /〇〇 でよいか？」と一度だけ確認する。

2. **内容の確定**
   - ユーザーが本文や手順を貼っている場合はそれをベースにする。
   - 概要だけの場合は、既存の `pre-review.md` や `github-issue-organize.md` のトーンに合わせて、制約・手順・フォーマットを書く。

3. **ファイル形式（既存コマンドに合わせる）**
   ```markdown
   ---
   description: （一言で何をするコマンドか）
   globs: ["**/*"]
   ---

   # /{コマンド名}

   （コマンドが実行されたときの役割を1行で）

   ## 制約（あれば）

   - …

   ## 実施手順

   1. …
   2. …
   ```
   - 既存の `~/dotfiles/claude/commands/*.md` と同じく、YAML frontmatter（description, globs）＋ 見出し `# /コマンド名` ＋ 本文。

4. **保存**
   - **保存先はデフォルトのみ使う。** 「どこに保存する？」と聞かず、`~/dotfiles/claude/commands/{コマンド名}.md` に作成または上書きする。
   - ユーザーが「リポジトリの .cursor に」「ここには保存しないで」などと**明示した場合だけ**、その指示に従う。

5. **完了報告**
   - 保存したパスを一行で伝える（例: `~/dotfiles/claude/commands/foo-bar.md` に保存しました）。
   - 必要なら「編集したいときはこのファイルを開いてください」と添える。

## 注意

- 新規フォルダは作らない。既存の `~/dotfiles/claude/commands/` にだけファイルを追加する。
- ソースコード（プロジェクトの実装）は編集しない。変更するのは dotfiles のコマンド .md のみ。
