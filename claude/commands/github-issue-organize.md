# /github-issue-organize

`/github-issue-organize` が実行されたとき、**PRイシュー整理アシスタント**として以下を実施する。

## 制約（必ず守る）

- **ソースコードには一切触れない。** 既存のコード・設計書の編集・追加・削除は行わない。
- **出力ファイルは PR 対象にしない。** 出力先のファイル名は必ず `review` から始める（例: `review-pr-1495.md`）。このファイルはリポジトリの `.gitignore` で除外し、PR に含めない。

## 実施手順

### 1. PR の未解決コメント取得

- ユーザーが PR 番号（例: 1495）または PR URL を指定している場合はそれを使用する。
- 未指定の場合は、現在のブランチから `gh pr view` で対象 PR を特定するか、ユーザーに PR 番号を確認する。
- `gh api repos/{owner}/{repo}/pulls/{pr_number}/comments` およびレビューコメント API を用い、**未解決（unresolved）または未返信**のコメント一覧を取得する。
- 取得できない場合は、ユーザーが貼ったコメント一覧や「未解決コメントを貼ってください」と案内して代替とする。

### 2. 出力ファイルの作成・更新

- ファイル名: **`review` で始まる名前**（例: `review-pr-1495.md`）。PR 番号が分かれば `review-pr-{番号}.md` を推奨。
- 出力先は**作業中のリポジトリのルート**（ユーザーが指定した場合はそのパス）。同じファイルが既にある場合は、未解決コメント一覧を最新化して上書きまたは追記する。

### 3. 出力フォーマット

以下の Markdown を出力する。**各コメントごとに「回答欄」と「対応コミット欄」を必ず用意する。**

```markdown
# PR 未解決コメント整理（PR #〇〇）

- PR: （URL または番号）
- 作成日: （日付）
- 対象ブランチ: （例: docs/trial-plan-report-management）

---

## 未解決コメント一覧

### コメント 1

- **ファイル:** （パス）
- **行:** （行番号）
- **投稿者:** （ユーザー名）
- **内容:**
  > （コメント本文）

**回答:**
<!-- ここにユーザーが回答を記入 -->

**対応コミット:**
<!-- 例: 〇〇コミットで対応 → ユーザーが記入 -->

---

### コメント 2

（同様のブロックを繰り返す）

---

## 対応サマリー（後から記入可）

| # | ファイル | 対応方針 | 対応コミット |
|---|----------|----------|--------------|
| 1 |          |          |              |
| 2 |          |          |              |
```

- 未解決コメントが 0 件の場合は、「未解決コメントはありません」とだけ記載した同じ形式のファイルを出力する。

### 4. ユーザーへの案内

- 出力ファイルパスを伝える。
- 「各コメントの『回答』と『対応コミット』を記入してください。記入後、対応するコミットを実施する場合は別途指示してください。」と案内する。

## .gitignore の確認

- 出力ファイルが PR に含まれないよう、`review*.md` または該当ファイル名が**リポジトリの** `.gitignore` に含まれていることを確認する。含まれていなければ、ユーザーに「`review*.md` を .gitignore に追加することを推奨します」とだけ伝える（**ソースコードを変更しない**ため、実際の .gitignore 編集はユーザーが行う）。
