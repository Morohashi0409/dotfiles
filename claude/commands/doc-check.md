# /doc-check

`/doc-check` が実行されたとき、**ドキュメント整合性チェッカー**として
コードと関連ドキュメントの乖離を検出し、更新案を提示する。

コード変更後に自動的に呼び出されることを想定しており、
`depends-on` 宣言のあるファイルとの整合性を優先的に確認する。

## 制約

- ドキュメントを無断で書き換えない。変更案を提示し、承認を得てから反映する
- 変更範囲は `depends-on` に宣言されたファイルと、直近の git 差分に関連するドキュメントに限定する
- スコープ外のドキュメントは触らない

## 実施手順

1. **直近の変更を把握する**
   ```bash
   git diff --name-only HEAD~1 HEAD 2>/dev/null || git diff --name-only --cached
   ```
   変更されたファイル一覧を取得する。

2. **depends-on 宣言を収集する**
   以下のパターンで `depends-on` を持つファイルを検索する：
   ```bash
   grep -r "depends-on:" ~/dotfiles/claude/commands/ --include="*.md" -l
   grep -r "depends-on:" .claude/ --include="*.md" -l 2>/dev/null
   ```

3. **ドキュメントの整合性をチェックする**
   変更されたファイルに関連するドキュメントを対象に、以下を確認する：

   - **CLAUDE.md**: スキル一覧・手順・ガイドラインが最新か
   - **README.md**: セットアップ手順・機能説明が変更に追随しているか
   - **コマンド .md ファイル**: 実装と手順が一致しているか
   - **API ドキュメント**: エンドポイント・引数・レスポンスに変更がないか

4. **乖離レポートを出力する**
   以下の形式でレポートを提示する：
   ```
   ## ドキュメント差分レポート

   ### 要更新
   | ファイル | 問題 | 提案 |
   |---------|------|------|
   | CLAUDE.md:45 | `/handover` スキルの説明が古い | 「...」→「...」に更新 |
   | README.md:12 | セットアップ手順に `mise install` が抜けている | 手順3を追加 |

   ### 問題なし
   - docs/architecture.md: 変更なし・整合性OK
   ```

5. **承認を得て更新する**
   - 「上記の更新を適用しますか？」と確認する
   - 承認を得たファイルのみ更新する
   - 更新後に再度チェックし、残存する問題がないか確認する

6. **完了報告**
   ```
   doc-check 完了:
   - チェック対象: {N} ファイル
   - 更新済み: {M} ファイル
   - 問題なし: {K} ファイル
   ```

## 自動発火のトリガー（depends-on による連携）

他のスキルの `depends-on` に `CLAUDE.md` が含まれている場合、
そのスキル実行後に doc-check が自動的に推奨される。

手動でも任意のタイミングで実行できる：
```
/doc-check
```

特定ファイルのみチェックしたい場合：
```
/doc-check README.md
/doc-check CLAUDE.md docs/setup.md
```
