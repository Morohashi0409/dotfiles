# RASHISA デモテスト参照ガイド

## Overview

`npm run seed:demo` で投入される 5テンプレート × 6シナリオ = **30プロジェクト**を使った手動テストの参照ガイド。各シナリオで何を確認すべきかを素早く引ける。

## デモデータのリセット

```bash
# .dev.vars が設定されている前提
DATABASE_URL="$(grep DATABASE_URL .dev.vars | cut -d= -f2-)" npm run seed:demo
```

テンプレートマスタが消えた場合は先に実行:
```bash
DATABASE_URL="..." npm run db:seed
```

## 30シナリオ一覧

| シナリオ | responses | answers | analyses | status |
|---------|-----------|---------|----------|--------|
| `[X] 空（回答なし）` | 0 | 0 | なし | accepting |
| `[X] 進行中（分析なし）` | 他者2名・前半50%のみ | 半答 | なし | accepting |
| `[X] 自己評価のみ` | 自己1件 | 全問 | なし | accepting |
| `[X] 他者評価のみ` | 他者3名 | 全問 | なし | accepting |
| `[X] 完了（3名）` | 自己+他者3名 | 全問 | あり | completed |
| `[X] 完了（10名）` | 自己+他者10名 | 全問 | あり | completed |

`[X]` = MBTI / ラブタイプ / 強み弱み / 社会人360 / エンジニア360

## テストケース一覧

### シナリオ: 空（回答なし）

**目的**: プロジェクト作成直後の初期状態

確認画面: `/projects/[id]/share`
- [ ] URL共有ボタンが表示される
- [ ] 回答数 0 と表示される
- [ ] 「分析する」ボタンが表示されない or 非活性

---

### シナリオ: 進行中（分析なし）

**目的**: 一部の回答者が途中まで回答した状態

確認画面: `/projects/[id]/share` または `/dashboard`
- [ ] 回答数 2 と表示される
- [ ] 分析ボタンが表示される（回答があるため）
- [ ] 結果画面に遷移しても分析未実施の案内が出る

---

### シナリオ: 自己評価のみ

**目的**: ホスト本人だけが回答した状態（他者未回答）

確認画面: `/answer/[id]/form`（自己回答フォームの表示確認）
- [ ] 回答数 1（自己評価）と表示される
- [ ] 自己回答が入力済み状態で再回答できない or 編集できる
- [ ] 他者評価が 0 であることが確認できる

---

### シナリオ: 他者評価のみ

**目的**: ゲストのみ回答し、ホストが自己評価していない状態

確認画面: `/projects/[id]/share`
- [ ] 回答数 3（全員ゲスト）と表示される
- [ ] ホスト自身の自己評価がない状態で分析実行できるか確認
- [ ] 結果でレーダーチャートが自己評価なしでも描画されるか

---

### シナリオ: 完了（3名）

**目的**: 少人数での結果表示の確認

確認画面: `/projects/[id]/result`
- [ ] レーダーチャートが描画される
- [ ] AI サマリー（business_title など）が表示される
- [ ] 「1番の理解者」セクションが表示される
- [ ] コメント一覧に田中さん・佐藤さん・鈴木さんが表示される
- [ ] ギャップ（自己 vs 他者）が視覚的に確認できる

テンプレート別追加確認:
| テンプレート | 確認ポイント |
|-------------|-------------|
| MBTI | self_type=INFJ / peer_type=ENFJ のギャップ表示 |
| ラブタイプ | 情熱型→包容型のギャップコメント |
| 強み弱み | business_title「縁の下のアーキテクト」・自己PR表示 |
| 社会人360 | トリセツ（skills/energy_source/caution）表示 |
| エンジニア360 | 隠れた才能（コード品質）表示 |

---

### シナリオ: 完了（10名）

**目的**: 大人数での結果・パフォーマンス確認

確認画面: `/projects/[id]/result`
- [ ] 完了（3名）と同じ確認 + 以下を追加
- [ ] 回答者数 10名 + 自己が正しく表示される
- [ ] レーダーチャートが10名分の平均で崩れない
- [ ] コメント一覧が全10名分スクロールできる
- [ ] ページ読み込みが著しく遅くない（10名データ）

## テンプレート横断チェック

すべてのテンプレートで「完了（3名）」を開いて以下を確認:

```
/projects/[MBTI完了3名のID]/result
/projects/[ラブタイプ完了3名のID]/result
/projects/[強み弱み完了3名のID]/result
/projects/[社会人360完了3名のID]/result
/projects/[エンジニア360完了3名のID]/result
```

各画面で:
- [ ] クラッシュなく表示される
- [ ] テンプレート名が正しく表示される
- [ ] axis_scores_json がレーダーチャートに反映されている

## 回答フォームのテスト

`/answer/[id]` のゲストウェルカム画面から:
- [ ] scale_5 の設問（強み弱み・社会人360・エンジニア360）: 1〜5 が選択できる
- [ ] scale_6 の設問（MBTI・ラブタイプ）: 0〜5 の6段階が動作する
- [ ] 自由記述（text）: 未入力でも送信できる（任意）
- [ ] 全設問未回答では「送信する」が押せない

> scale_5 テンプレートには「強み弱み」の空プロジェクトを、scale_6 テンプレートには「MBTI」の空プロジェクトを使うと確認しやすい。

### 「進行中」シナリオの半答確認

`[X] 進行中（分析なし）` は前半50%のみ回答済み。Drizzle Studio または DB で:
```sql
SELECT COUNT(*) FROM answers a
JOIN responses r ON r.id = a.response_id
JOIN projects p ON p.id = r.project_id
WHERE p.name LIKE '%進行中%';
```
→ 全問数の50%（切り捨て）× 回答者2名 が期待値

## よくあるテストフロー

### 新機能実装後の基本確認（5分）
1. `npm run seed:demo` でリセット
2. `/dashboard` で30件表示を確認
3. 「完了（3名）」を1テンプレート開いて結果画面を確認

### 結果画面の全テンプレート確認（15分）
1. `npm run seed:demo` でリセット
2. 5テンプレート × 完了（3名）を順番に開く
3. 各画面でクラッシュ・崩れがないか確認

### エッジケース確認（10分）
1. 「空（回答なし）」 → 回答フォームから実際に回答してみる
2. 「進行中」 → 分析を手動トリガーして結果が出るか確認
3. 「他者評価のみ」 → 自己評価なしでの結果表示確認

## デモデータの前提条件

| 前提 | 確認方法 |
|-----|---------|
| テンプレートマスタが存在する | `npm run db:seed` で投入済みか確認 |
| `DEV_BYPASS_AUTH=true` が設定済み | `.dev.vars` を確認 |
| `DATABASE_URL` が設定済み | `.dev.vars` を確認 |

## 開発環境前提

- `DEV_BYPASS_AUTH=true` が `.dev.vars` に設定済み（dev-test-user として動作）
- `npm run dev` でローカルサーバー起動
- デフォルト URL: `http://localhost:5173`

## デモデータが対応しないケース（別途テスト必要）

- `is_anonymous=true`（匿名表示: コメント欄が「匿名ユーザー1」になる）
- `allow_respondents_view_result=true`（回答者が結果を閲覧できる）

これらは現在のデモデータでは `false`（デフォルト）。機能テスト時は手動で DB を更新するか、プロジェクト作成画面から設定する。
